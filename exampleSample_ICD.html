This is a sample protocol made up to demonstrate features of the PolyPacket code generation tool. The idea is to have a tool that can automatically create parseable/serializable messaging for embedded systems.

Packet Types

Ping

  • Packet ID: [00]
  • Requests: Ack
This Packet type does not contain any data fields

Ack

  • Packet ID: [01]
  • Responds To: Ping
This Packet type does not contain any data fields

SendCmd

  • Packet ID: [02]

Message to send command to node


Structure:
Byte 0
Field cmd
Type uint8_t

Fields:
Field Description
cmd command byte for controlling node
  • 0x00 : LED_ON - turns on led
  • 0x01 : LED_OFF - turns off led
  • 0x02 : RESET - resets the device

GetData

  • Packet ID: [03]
  • Requests: Data

Message to get data from node


This Packet type does not contain any data fields

Data

  • Packet ID: [04]
  • Responds To: GetData

Message containing data from sensor


Structure:
Byte 0 1 2 3 4 5 6 ........ 38
Field sensorA sensorB sensorName
Type int16_t int char [32]

Fields:
Field Description
sensorA Value of Sensor A
sensorB Value of Sensor B
sensorName Name of sensor responding to message
